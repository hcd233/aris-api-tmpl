# MVC to DDD Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 参考 `../aris-proxy-api`，将当前用户、令牌、OAuth2 请求链路从 MVC service 直连 DAO 迁移为 DDD 分层。

**Architecture:** 新增 `domain`、`application`、`infrastructure/repository`，由 `bootstrap` 作为组合根通过 `dig` 注入依赖。`handler` 只做 DTO 与 command/query 的适配，`router` 只注册路由，`application` 编排领域对象和仓储接口，`infrastructure` 负责 GORM/对象存储/OAuth2 实现。

**Tech Stack:** Go 1.25.1、Fiber、Huma、`go.uber.org/dig`、GORM、标准库 `testing`。

---

## 任务

### Task 1: 建立 DDD 骨架

**Files:**
- Create: `internal/domain/common/aggregate/root.go`
- Create: `internal/domain/identity/...`
- Create: `internal/domain/oauth2/...`
- Create: `internal/application/identity/...`
- Create: `internal/application/oauth2/...`
- Create: `internal/infrastructure/repository/...`

**Steps:**
1. 定义聚合根、值对象、领域仓储接口。
2. 将用户查询、用户更新、刷新 token、OAuth2 登录/回调拆为 command/query handler。
3. 使用基础设施仓储实现领域接口。

### Task 2: 改造接口层

**Files:**
- Modify: `internal/handler/*.go`
- Modify: `internal/router/*.go`

**Steps:**
1. `handler` 构造函数接收 application 接口，不再创建 service。
2. `router` 接收 handler 依赖，不再创建 handler/service。

### Task 3: 改造组合根

**Files:**
- Modify/Create: `internal/bootstrap/server.go`
- Create: `internal/bootstrap/router.go`
- Modify: `cmd/server.go`

**Steps:**
1. 使用 `dig` 注册 HTTP、repository、application、handler provider。
2. `BuildServer() (*Server, error)` 返回启动错误。
3. `RegisterRoutes(*Server) error` 从容器解析 handler 并注册路由。

### Task 4: 移除 MVC service

**Files:**
- Delete: `internal/service/*.go`

**Steps:**
1. 确认没有 `internal/service` 引用。
2. 删除旧 service 包，防止新旧架构并存。

### Task 5: 验证

**Commands:**
- `gofmt`/`go test -count=1 ./...`
- 必要时 `make lint`
